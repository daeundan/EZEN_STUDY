시멘틱 태그를  보기 쉽게 정리한 이미지 자료 (출처: https://www.openads.co.kr/content/contentDetail?contsId=11964)

시맨틱 태그 : HTML의 구조를 설계하는데 있어 태그에 의미를 부여함으로써 웹사이트의 구조를 파악하기 쉽도록 도와주기 위해 만들어진 것

내가 느끼기로는, div 남발으로 인한 구조 파악의 어려움 + 코드를 더욱 명료하게 찾을 수 있도록(가독성) 하기 위해 만든 이름인 것 으로 보인다.

시멘틱 태그의 종류 : <header>, <nav>, <article>, <section>, <footer>, <main> 등

MDN 에서 나열된 Semantic Tac의 요소는 100개 이상 가량 되는데, 그 중 대표적인 10개는 다음과 같다.
<article>, <details>, <figcaption>, <figure>, <footer>, <form>, <header>, <main>, <mark>, <nav>, <section>, <summary>, <time>

기억이 가물가물 하지만, 주된 태그들은 수업 시간에 활용해 본 듯 하다.

main, nav, footer 는 주로 상위 목록을 묶어서 구분할 때  사용하였다.
header > section > article - 주로 각 파트별로 내용이 상이한 단락들을 구분할 때 사용하였다.
detail 은 상세 내용을 적어야 할 때 사용하였다
그 안에 form 태그를 넣어 input이나 내용을 넣기도 하였다.
figure 는 video 태그를 넣을 때 사용하였다.

mark 나 summary 는 아직 사용하지 않은 것 같다.

확인해보니, 상세 기능은 다음과 같다.

기능 : 의미 없는 div 태그가 아닌 더욱 직관적인 단어의 사용 단락을 구별하여 일반적인 HTML 태그들의 순서, 위치 파악에 도움을 줌

1) 웹사이트의 일관성 향상(접근성)

2) 태그 식별과 해석이 용이함(가독성)

3) 검색 엔진의 풍부함 구조화된 코드(SEO 향상)


**시멘틱 태그 사용 측면에서 정리**

<article> 독립적인 글 
ex) 블로그 게시물, 뉴스 기사, 제품 리뷰 등 독립적으로 배포하거나 재사용할 수 있는 독립형 콘텐츠를 정의

 
<aside> 옆 콘텐츠 담당
ex) 페이지 콘텐츠를 제외한 콘텐츠를 정의. 문서 중 사이드 바에 해당됨.

<details> 사용자가 숨길 수 있는 세부 콘텐츠 게시
사용자와 상호작용 가능. 버튼 통해 열고 닫음. 
기본적으로 닫은 상태에서 click = > (내용이 보임)

<details> 태그는 <summary>와 친함 = 보통 같이 사용한다는 의미
details 에서 보이는 부분 담당!
summary는 detail 태그 첫번째 하위항목이어야 함

<figure> & <figcaption>

<figure> 일러스트레이션, 다이어그램, 사진, 코드 목록 등과 같은 자체 포함된 콘텐츠를 지정

<figcaption> 태그는 <figure> 요소에 대한 캡션 정의, 사진의 설명 위해 사용
Caption = 이미지에 대한 간략한 설명 => <figcaption>은 <figure>에 대한 간략한 설명

<footer> 바닥에 있는 글. 맨 하단에 위치한 정보들..
ex) 연락처 정보, 사이트 맵, 웹사이트를 하나로 묶고 SEO를 강화하는 데 도움이 되는 소셜 미디어 사이트에 대한 링크 등

<header> 문서나 섹션의 머릿글을 지정하며, 로고, 탐색, 제목 및 기타 소개 정보가 포함된 페이지 상단 부분을 정의
ex) 메타 태그 정보, 키워드, CSS 파일이나 스타일 시트도 포함되는 경우가 많음.

<main> 메인 내용을 담는 태그
웹사이트의 텍스트 본문이나 콘텐츠. 문서에서 유일해야함.
<article>, <aside>, <footer>, <header>, <nav> 등 모든 페이지의 태그 앞에 자리함.

<nav> 네비게이션 역할 탐색할 수 있도록 특정 섹션 구성함.
웹사이트의 메뉴, 탭, 탐색경로 등 탐색 링크가 포함된 페이지 부분을 정의

<section> 문서의 부분을 의미하는 태그
기본 콘텐츠 내의 특정 주제 또는 부제목과 관련된 주제별 콘텐츠 그룹을 정의.
<section> 안에 <section>을 넣을 수도 있고, <article>을 이용해 내용을 넣을 수 있음
